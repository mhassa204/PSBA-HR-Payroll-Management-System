# PSBA HR Management System - User Manual

## Part 1: Introduction and Getting Started

---

## Welcome to PSBA HR Management System

The PSBA HR Management System is a comprehensive human resource management platform designed to streamline employee management, attendance tracking, leave management, travel requests, and organizational administration for the Punjab Small Business Association.

This manual will guide you through using the system to manage your daily HR tasks efficiently.

---

## System Access and Login

### Accessing the System

1. Open your web browser (Chrome, Firefox, Edge, or Safari)
2. Navigate to the PSBA HR Management System URL provided by your IT administrator
3. The login page will display with fields for Email and Password

### Login Process

1. **Enter Your Email Address**

   - Type your official email address in the "Email" field
   - This is the email address registered in the system by your administrator
2. **Enter Your Password**

   - Type your password in the "Password" field
   - The default password for all new users is: `abc123`
   - You are strongly advised to change this password after first login (contact IT support for password changes)
3. **Click Login**

   - Click the "Login" button to access the system
   - If your credentials are incorrect, you will see an error message
4. **What Happens After Login**

   - Upon successful login, you will be redirected to the Dashboard or Home screen
   - The system will show you different options based on your user role and permissions
   - Your profile information and assigned permissions will be loaded automatically

### Logging Out

1. Click on your profile/name at the top of the screen
2. Click the "Logout" button
3. You will be logged out and redirected to the login page

---

## Understanding User Types

The PSBA HR system supports three distinct user types, each with specific access and responsibilities:

### 1. Employee-Based Users

These are individual employees who have personal accounts linked to their employee records.

**Characteristics:**

- Linked to a specific employee profile in the system
- Can view and manage their own information
- Can apply for leaves and travel requests
- Can view their own attendance records
- May have approval responsibilities for subordinates
- Access is personalized and limited to their own data

**Common Roles:**

- Regular Employees
- Assistant Directors
- Managers
- Directors

**What They Can Do:**

- Apply for leaves and travel
- View personal information
- Submit travel expense claims
- View attendance history
- Approve requests from subordinates (if applicable)

---

### 2. Department-Based Users

These are users linked to specific departments or administrative sections.

**Characteristics:**

- Linked to a particular department (e.g., Operations, Establishment, Accounts)
- Handle department-wide responsibilities
- May manage approvals for their department
- Can access and process requests originating from their department

**Common Roles:**

- Operations Staff
- Accounts Personnel
- Establishment Officers
- HR Staff

**What They Can Do:**

- Process department-specific tasks
- Approve requests within their domain
- Manage department activities
- View department-related data
- Handle administrative workflows

**Special Departments:**

- **Operations Department:** Approves travel requests from bazaar locations
- **Accounts Department:** Processes expense claims and manages financial transactions
- **Establishment Department:** Verifies and manages employee records and documents

---

### 3. Location-Based Users

These are users linked to specific physical locations (Head Office or Bazaars).

**Characteristics:**

- Linked to a specific location (e.g., "Head Quarter Lahore" or "Sahulat Bazaar Mian Plaza")
- Handle location-specific tasks and management
- Can create duty rosters for their location
- Manage attendance and local operations

**Common Roles:**

- Location Managers
- Bazaar In-Charges
- Head Office Coordinators

**What They Can Do:**

- Create and manage duty rosters
- View attendance data for their location
- Manage location-specific settings
- Handle local staff assignments
- Monitor location activities

---

## Understanding Your Permissions

Your access to system features depends on your **User Type** and your assigned **Role**. The system uses role-based access control to ensure you only see and can perform actions appropriate to your position.

### How Permissions Work

- **View Permission:** Allows you to see and read information
- **Create Permission:** Allows you to add new records
- **Update Permission:** Allows you to modify existing records
- **Delete Permission:** Allows you to remove records
- **Approve Permission:** Allows you to approve/reject requests

### Common Permission Categories

- **Employee Management:** View, create, update employee records
- **Attendance:** View and manage attendance data
- **Leave Management:** Apply for leaves or approve leave requests
- **Travel Management:** Submit travel requests or approve travel claims
- **Settings:** Configure system settings (admin only)
- **Reports:** Generate and view reports
- **Users:** Manage system users (admin only)

If you need access to a feature you cannot see, contact your system administrator.

## Overview

The Attendance Management module provides comprehensive attendance tracking using biometric devices. This module allows you to monitor employee attendance, manage devices, view attendance by location, and generate attendance reports.

**Key Features:**

- Biometric device integration
- Location-based attendance tracking
- Attendance by employee
- Device management
- Attendance history and reports

**Who Can Use This Module:**

- HR Staff (Establishment role)
- Location Managers
- Users with attendance permissions

---

## Understanding Attendance Structure

### Locations

The system organizes attendance by **Locations**:

- **Head Office:** Main administrative office
- **Bazaars:** Individual sales locations (e.g., "Sahulat Bazaar Mian Plaza")

Each location has:

- Assigned employees
- Biometric devices
- Opening and closing times
- Manager(s)

### Devices

Biometric devices are the hardware used to record attendance:

- Installed at specific locations
- Connected to the network
- Capture employee check-in/check-out times
- Automatically sync data to the system

### Attendance Records

Each attendance entry contains:

- Employee information
- Date and time
- Type (IN/OUT - check-in or check-out)
- Device information
- Location

---

## Viewing Attendance by Location

### Accessing Location List

1. From the main menu, click **"Attendance"**
2. Click **"Locations"**
3. You'll see a list of all locations

### Location Details

The location list shows:

- Location name
- Type (Head Office or Bazaar)
- District and City
- Status (Active/Inactive)
- Number of employees assigned
- Number of devices installed

### Opening a Location

1. Click on a location name

   - **LSR:** Late Stay Report

## Viewing First Morning On-duty (FMO)

### What is FMO?

FMO shows who reported first at each location each day. This helps:

- Monitor early arrivals
- Track punctuality
- Ensure locations open on time

### Accessing FMO Report

1. Open a location
2. Click the **"FMO"** tab

4. View the report

### Understanding FMO Report

The report shows:

- Date
- Employee name

- Location
- Any remarks

---

## Viewing Duty Rosters

### What is a Roster?

A roster is the scheduled duty assignment for employees at a location. It defines:

- Which employees work on which days
- Working hours for each day
- Time slots and breaks

### Accessing Roster Information

1. Open a location
2. Click the **"Roster"** tab
3. View roster for the selected period

### Understanding Roster Display

The roster shows:

- Employee names
- Day-by-day schedule
- Time slots (from/to)
- Location assignments
- Weekly off days

---

## Viewing Late Stay Report (LSR)

LSR identifies employees who stayed late (beyond normal hours). This helps:

- Ensure safety compliance

### Accessing LSR Report

## Understanding Expense Claim Calculations

### TADA System

The system calculates TADA (Travel Allowance and Daily Allowance) based on:

1. **Travel Allowance:**

   - Calculated from distance and rate per km
   - Based on your scale grade
   - Rate per km varies by grade
2. **Per Diem:**

   - Daily allowance rate
   - Based on your scale grade
   - Applied only for overnight stay
   - Calculation: Days × Per Diem Rate
3. **Toll Tax:**

   - Toll receipts if any
   - Added to travel allowance

### Transport Mode Selection

#### Own Vehicle

- Select if using personal vehicle
- System calculates distance allowance
- Rate per km applied
- Toll tax can be added

#### Other Transport

- Select if using other transport (bus, train, etc.)
- System calculates fare amount
- Add actual fare paid
- No per km calculation

### Grade-Based Rates

Rates are predefined by scale grade:

- **BPS-17:** Base rates
- **BPS-18:** Higher rates
- **BPS-19:** Higher rates
- **BPS-20:** Highest rates

These rates are configured by the Accounts department.

## Overview

## Understanding Duty Rosters

### What is a Roster?

A duty roster is a work schedule that specifies:

- **Which employees** work on which days
- **What times** they work each day
- **Where** they are assigned (if multiple locations)
- **Special assignments** or notes

### Roster Terms

- **Schedule Period:** The time range the roster covers (e.g., "December 2024")
- **Employee Entry:** One employee's schedule for the roster period
- **Day Schedule:** The schedule for one day of the week
- **Time Slot:** Start and end times for a shift
- **Weekly Off:** Days when an employee does not work

### Approval Status

Rosters have approval statuses:

- **Pending:** Awaiting approval
- **Approved:** Approved and active
- **Rejected:** Not approved

---

## Creating a New Duty Roster

   - Example: "Security Roster - December 2024"
   - This helps identify rosters later
2. **Valid From**

   - Select start date from calendar
   - First day the roster becomes effective
3. **Valid To**

   - Select end date from calendar
   - Last day the roster is effective
   - Ensure it's after the start date

#### Step 2: Understanding the Roster Interface

- **Role Tags:** Grouped by job category
- **Employee Name:** Individual employees

For each employee, you'll see a grid with:

- **Day Columns:** Monday through Sunday

   - **Time From:** Start time (e.g., 09:00)
   - **Time To:** End time (e.g., 17:00)
   - **Location:** Optional location identifier

1. Click on the day cell
2. Select schedule type as "Weekly Off"
3. That day is marked as off for the employee

1. Click on the day cell
2. Set schedule type to "Half Day"
3. Enter partial time:

1. Click on the day cell
2. Select schedule type as "Leave"

1. Find the employee row

3. Enable collective weekly off
4. Set:
   - **From Day:** First day of off (e.g., Saturday)

## User Management

### What is User Management?

User management controls system access and accounts.

### Accessing User Management

1. Go to Users from main menu
2. View all system users

### Understanding Users

The user list shows:

- Email (username)
- Role
- Linked Employee
- Status (Active/Inactive)
- Department (if department-based)
- Location (if location-based)

### Creating a New User

1. Click **"Add User"** button
2. Select type:
   - **Employee-Based:** Link to employee
   - **Department-Based:** Link to department
   - **Location-Based:** Link to location
3. Fill in:
   - **Email:** User's email address
   - **Password:** Default password (abc123)
   - **Role:** Select appropriate role
   - **Link:** Link to employee/department/location
4. Click **"Save"**

### Types of Users

#### Employee-Based User

- Linked to specific employee
- Accesses their own information
- Can apply for leaves/travel

#### Department-Based User

- Linked to specific department
- Handles department tasks
- Can approve department items

#### Location-Based User

- Linked to specific location
- Manages location-specific data
- Can create rosters for location

### Managing User Access

1. Find user in list
2. Click **"Edit"**
3. Update:
   - Role
   - Status (enable/disable)
   - Link information
4. Save changes

### Disabling Users

To deactivate a user account:

1. Open user record
2. Update status to inactive
3. Save changes

**Note:** Don't delete users, disable them to preserve history.
